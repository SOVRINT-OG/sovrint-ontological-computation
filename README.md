## Mandate

This repository defines the public formal architecture of **Ontological Computation™** within SOVRINT™. It treats entities, relations, states, transitions, constraints, observations, and corrections as computable structural objects.

Ontological Computation™ is not a replacement for conventional computation. It is a higher-order systems layer used to specify what a system contains, how its components relate, what changes are admissible, what remains invariant, and how meaning is preserved across transformations.

## Core Scope

- ontological primitives
- relational topology
- state and transition classes
- structural operators
- semantic invariants
- cross-register translation
- constraint propagation
- correction and reconciliation logic
- observability bindings
- governance bindings
- public examples and formal specifications

## Foundational Axiom

Reality is not merely layered. It is threaded through relations operating beneath, above, around, through, within, between, across, and beyond observable objects.

This repository formalizes those relations without reducing them to metaphor-only language.

## Canonical Boundaries

This public repository may contain approved theory, formal definitions, diagrams, examples, schemas, and publication-ready specifications.

It must not contain protected runtime code, unreleased equations, private datasets, credentials, internal security architecture, or unpublished engine internals.
